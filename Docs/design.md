# Neural Routing

## Концепция

Пазл под Ludum Dare 59, тема **Signal**.

Поле — квадратная сетка из цветных плиток. Юнит стоит на старте и смотрит вправо. У него есть «мозг» из трёх входных нейронов (Red / Yellow / Blue) и трёх выходных (Forward / Up-diag / Down-diag). Игрок тянет провода между входами и выходами. На Play юнит шагает тик за тиком: читает цвет плитки под собой, по проводам сигнал уходит в выход, юнит шагает в ту сторону. Цель — дойти до выхода, не наступив на стену.

## Нарратив и сеттинг (черновик)

**Герой.** Робот-курьер на парящей гидравлической подушке — держится над землёй на небольшой высоте, может дрифтить вперёд и по диагоналям. Ни влево, ни назад — геометрия подушки не позволяет.

**Место действия.** Старые мосты над пропастью / туманом / водой. Мост длинный, в некоторых секциях дыры (плитки `.`), в других — завалы / обломки (плитки `#`), которые надо обходить.

**Сигнал — двойная метафора** (объединение двух трактовок «signal»):
1. *Внутренний сигнал.* Провод от сенсора к мотору — тот самый сигнал, который игрок маршрутизирует в «мозге» робота. Это тот слой, что у нас уже реализован механически.
2. *Внешний сигнал.* Цветные плитки — это наведённые электромагнитные / радио помехи в зонах моста, которые ловит сенсор робота. Красные, зелёные и синие секции — разные типы сигналов. Игрок не управляет движением напрямую, он заранее **программирует реакцию** робота на каждый тип сигнала.

Так тема «Signal» просвечивает и в геймплее (игрок буквально прокладывает провода сигналов), и в сеттинге (робот едет по полосе сигналов, и сам несёт сигнал).

**Мотивация** (выбрать один вариант, TBD):
- Связь на той стороне потеряна, робот — запасной ретранслятор, везёт свой «сигнал».
- SOS: на другой стороне кто-то застрял, робот — разведчик.
- Последнее сообщение: робот несёт данные, которые больше некому доставить.

**Визуальные наметки** (для арта, не фиксировано):
- Мост — ржавое железо, дерево, решётки; под ним туман/пропасть.
- Плитки — секции моста с пульсирующим цветовым подсвечиванием (сигнал наложен на поверхность).
- Робот — небольшая коробочка с антенной, голубое сияние снизу (подушка).
- Плата мозга — «диспетчерский пульт»: нейроны-лампочки, провода — кабели, при активности искрят.
- Победа: «signal received», короткая вспышка на той стороне.

Сеттинг задаёт направление для арта, звука и музыки. В механике ничего не меняет.

## Что уже сделано

- **Меню → Level Select → Game.** MainMenu с Play/Music/SFX, LevelSelect с кнопками по JSON-манифесту, Game с переключением между уровнями через попап.
- **Модель.** `TileColor`, `NeuronNode` (+ `InputNode`, `OutputNode`, `NodeCatalog`), `Wire`, `BrainData`, `LevelData`, `UnitState`.
- **Симулятор.** `Simulator.Simulate(level, brain)` → `List<MoveCommand>` с полями `From`, `To`, `Status`, `Highlights`.
- **JSON-лоадер.** `LevelLoader.LoadFromResources(id)` через Newtonsoft.Json.
- **Рендер поля.** Плитки и юнит как UI `Image`, координаты логически горизонтальные (+X forward), визуально повёрнуты на 90° (снизу вверх).
- **Рендер мозга.** Нейроны-кружки слева (низ = входы, верх = выходы), провода — растянутые Image между точками.
- **Интерактив проводов.** Drag с входа → на выход создаёт провод; старый провод с того же входа удаляется; drop в пустоту = без провода.
- **Управление.** Play / Reset / Back / To List / Next; Fast-toggle (х3) с сохранением состояния в `GameSession.FastPlayback` (в рамках одного запуска).
- **Попап Win** с Next и To List. Crash/Stuck — через `statusLabel`, подсветка последнего тика остаётся на Crash/Stuck.
- **Подсветка на бегу.** Активный вход/провод/выход моргает (on во время шага, off во время паузы). Clear на Win/Reset/Play.
- **Уровни 01-05** в `Resources/Levels/` + `levels.json` манифест.

## Структура JSON-уровня

```json
{
  "id": "01",
  "name": "First Steps",
  "width": 10,
  "height": 5,
  "tiles": [
    "..........",
    "..........",
    "RRRRRRRRRR",
    "..........",
    ".........."
  ],
  "start": { "x": 0, "y": 2 },
  "exit":  { "x": 9, "y": 2 },
  "nodes": [
    ["In.R", "In.Y", "In.B"],
    ["Out.D", "Out.F", "Out.U"]
  ]
}
```

| Поле | Тип | Описание |
|---|---|---|
| `id` | string | Уникальный идентификатор уровня. |
| `name` | string | Название, показывается игроку. |
| `width` | int | Ширина сетки в клетках. |
| `height` | int | Высота сетки в клетках. |
| `tiles` | string[] | Массив строк длины `width`, всего `height` строк. Первая строка = верх сетки (`y = height - 1`), последняя = низ (`y = 0`). |
| `start` | {x, y} | Координаты стартовой клетки. |
| `exit` | {x, y} | Координаты клетки выхода. |
| `nodes` | string[][] | Массив столбцов. Каждый внутренний массив = столбец нейронов сверху вниз (или слева направо на плате). Обычно два столбца: входы и выходы. |

### Символы в `tiles`

| Символ | Значение |
|---|---|
| `.` | Пустота / пропасть. Шаг сюда — краш. |
| `#` | Стена. Шаг сюда — краш. |
| `R` | Красная плитка. |
| `Y` | Жёлтая плитка. |
| `B` | Синяя плитка. |

### Идентификаторы нейронов в `nodes`

Формат — `Type.Param`.

| ID | Тип | Описание |
|---|---|---|
| `In.R` | Вход | Активен, когда юнит стоит на красной плитке. |
| `In.Y` | Вход | Активен, когда юнит стоит на жёлтой. |
| `In.B` | Вход | Активен, когда юнит стоит на синей. |
| `Out.F` | Выход | Шаг вперёд `(+1, 0)`. |
| `Out.U` | Выход | Шаг по диагонали вверх `(+1, +1)`. |
| `Out.D` | Выход | Шаг по диагонали вниз `(+1, -1)`. |

Порядок в столбце = порядок отображения. Принятая компоновка выходов: `Out.D`, `Out.F`, `Out.U` — слева направо соответствует визуальным ↖, ↑, ↗.

## Бэклог

Отложено до появления арта и общего визуального направления:

- **Звук.** Клипы для кликов, шагов, победы, краша; фон-музыка в меню.
- **Арт.** Спрайты плиток, юнита, нейронов, проводов, фона; замена текущих placeholder-кружков и прямоугольников.
- **Покраска проводов по цвету входа** — отложено до арта (вероятно шейдерный эффект вместо простого tint).
- **Пульсация / glow на подсветке** — визуальная полировка тика.
- **Idle-покачивание плиток** («летающие площадки»).
- **Стилизация кнопок и UI** — фоны, скругления, шрифты.

Функциональные улучшения (можно сделать и до арта):

- **Подсказка на первом уровне** (tooltip / floating text: «Drag red to arrow»).
- **Клик по проводу — удалить.** Сейчас провод снимается только через drag.
- **Явная блокировка drag'а с выхода.** Сейчас работает через `is InputNode`-проверку, но стоит завернуть в `OnBeginDrag` и выключать обработчик заранее.
- **Счётчик проводов / тиков** — если решим вводить ограничения / ачивки.
- **Skip-to-next-level** в каком-то дебаг-меню, чтобы тестерам не приходилось проходить всё.
- **Показ решения в попапе Win** — маленькая визуализация, «ты решил так».
- **Сохранение прогресса.** Ludum Dare хостинг не дружит со storage — пока держим в `GameSession` (живёт до перезагрузки вкладки).
- **Phase 2 (стретч, не трогали):** колонка промежуточных нейронов (`And`, `Or`, `Not`, `Delay`), аналоговый сигнал «текущий 1.0 + прошлый 0.5», argmax на выходах.
