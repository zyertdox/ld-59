# Neural Routing

## Концепция

Пазл под Ludum Dare 59, тема **Signal**.

Поле — квадратная сетка из цветных плиток. Юнит стоит на старте и смотрит вправо. У него есть «мозг» из трёх входных нейронов (Red / Green / Blue) и трёх выходных (Forward / Up-diag / Down-diag). Игрок тянет провода между входами и выходами. На Play юнит шагает тик за тиком: читает цвет плитки под собой, по проводам сигнал уходит в выход, юнит шагает в ту сторону. Цель — дойти до выхода, не наступив на стену.

## Структура JSON-уровня

```json
{
  "id": "01",
  "name": "First Steps",
  "width": 10,
  "height": 5,
  "tiles": [
    "..........",
    "..........",
    "RRRRRRRRRR",
    "..........",
    ".........."
  ],
  "start": { "x": 0, "y": 2 },
  "exit":  { "x": 9, "y": 2 },
  "nodes": [
    ["In.R", "In.G", "In.B"],
    ["Out.F", "Out.U", "Out.D"]
  ]
}
```

| Поле | Тип | Описание |
|---|---|---|
| `id` | string | Уникальный идентификатор уровня. |
| `name` | string | Название, показывается игроку. |
| `width` | int | Ширина сетки в клетках. |
| `height` | int | Высота сетки в клетках. |
| `tiles` | string[] | Массив строк длины `width`, всего `height` строк. Первая строка = верх сетки (`y = height - 1`), последняя = низ (`y = 0`). |
| `start` | {x, y} | Координаты стартовой клетки. |
| `exit` | {x, y} | Координаты клетки выхода. |
| `nodes` | string[][] | Массив столбцов. Каждый внутренний массив = столбец нейронов сверху вниз. Обычно два столбца: входы и выходы. |

### Символы в `tiles`

| Символ | Значение |
|---|---|
| `.` | Пустота / пропасть. Шаг сюда — краш. |
| `#` | Стена. Шаг сюда — краш. |
| `R` | Красная плитка. |
| `G` | Зелёная плитка. |
| `B` | Синяя плитка. |

### Идентификаторы нейронов в `nodes`

Формат — `Type.Param`.

| ID | Тип | Описание |
|---|---|---|
| `In.R` | Вход | Активен, когда юнит стоит на красной плитке. |
| `In.G` | Вход | Активен, когда юнит стоит на зелёной. |
| `In.B` | Вход | Активен, когда юнит стоит на синей. |
| `Out.F` | Выход | Шаг вперёд `(+1, 0)`. |
| `Out.U` | Выход | Шаг по диагонали вверх `(+1, +1)`. |
| `Out.D` | Выход | Шаг по диагонали вниз `(+1, -1)`. |

Порядок в столбце = порядок отображения сверху вниз.
